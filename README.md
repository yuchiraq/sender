# Сервис рассылки

Веб-сервис на Go для постоянной рассылки писем по очереди. Сервис хранит очередь адресов, архив обработанных адресов, банк текстов, прикладывает файлы к письму и после успешной отправки переносит адрес в архив.

## Возможности

- Веб-панель полностью на русском языке.
- Темный современный интерфейс с отдельными блоками для очереди, банка текстов, вложений, журнала событий и безопасности.
- Фоновая отправка по таймеру и ручной запуск из панели.
- JSON-хранилища для очереди, архива, текстов и журнала.
- Авторизация, подписанные сессии, CSRF-защита, ограничение попыток входа и локальный доступ по умолчанию.
- Встроенная смена пароля администратора прямо из панели. Новый пароль сохраняется в `config.json` в виде `bcrypt`-хэша.
- Подготовленная конфигурация под Gmail через `smtp.gmail.com`.

## Важный нюанс по порту

Порт `80162` использовать нельзя, потому что диапазон TCP/UDP ограничен значениями `1..65535`. Поэтому сервис настроен на `127.0.0.1:60162`.

## Быстрый старт

1. Откройте [config.json](./config.json).
2. Укажите ваш адрес Gmail в `smtp.username` и `smtp.from_email`.
3. Задайте пароль приложения Gmail через переменную окружения:

```powershell
$env:SENDER_GMAIL_APP_PASSWORD="ваш_16_значный_пароль_приложения"
```

4. Если хотите заранее задать пароль администратора вручную, сгенерируйте `bcrypt`-хэш:

```powershell
go run ./cmd/sender hash-password "ваш-сильный-пароль"
```

5. Вставьте полученный хэш в `auth.password_hash`.
6. Положите файлы для отправки в папку [attachments](./attachments/).
7. Запустите сервис:

```powershell
go run ./cmd/sender
```

8. Откройте [http://127.0.0.1:60162](http://127.0.0.1:60162).
9. После входа при необходимости измените пароль администратора прямо в панели, в блоке `Смена пароля`.

## Полная настройка Gmail

### Что нужно подготовить в аккаунте Google

1. Включите двухэтапную аутентификацию для Google-аккаунта.
2. Создайте пароль приложения Google.
3. Используйте именно пароль приложения, а не обычный пароль от Gmail.

Официальные материалы Google:

- [Вход с помощью паролей приложений](https://support.google.com/accounts/answer/185833?hl=en)
- [Как отправлять почту с принтера, сканера или из приложения](https://support.google.com/a/answer/176600?hl=ru)

### Какие значения указывать в сервисе

Для Gmail используйте такие параметры:

- `smtp.host`: `smtp.gmail.com`
- `smtp.port`: `587`
- `smtp.security`: `starttls`
- `smtp.username`: полный адрес Gmail, например `your.name@gmail.com`
- `smtp.from_email`: обычно тот же адрес Gmail
- `smtp.password_env`: `SENDER_GMAIL_APP_PASSWORD`

Если хотите использовать SSL вместо STARTTLS:

- `smtp.port`: `465`
- `smtp.security`: `tls`

### Пример настройки из config.json

```json
"smtp": {
  "host": "smtp.gmail.com",
  "port": 587,
  "username": "your.name@gmail.com",
  "password": "",
  "password_env": "SENDER_GMAIL_APP_PASSWORD",
  "from_email": "your.name@gmail.com",
  "from_name": "Почтовый сервис",
  "security": "starttls",
  "insecure_skip_verify": false
}
```

### На что обратить внимание

- Без пароля приложения отправка через Gmail не пройдет.
- Если вы измените обычный пароль Google-аккаунта, ранее созданные пароли приложений могут перестать работать, и их нужно будет выпустить заново.
- Для рабочих или учебных аккаунтов Google Workspace администратор может ограничивать использование паролей приложений.
- Если в панели видно предупреждение, что Gmail настроен не полностью, проверьте `smtp.username`, `smtp.from_email` и `SENDER_GMAIL_APP_PASSWORD`.

### Проверка подключения

После запуска сервиса:

1. Войдите в веб-панель.
2. Убедитесь, что в блоке `Состояние системы` показано `Почта готова`.
3. Добавьте тестовый адрес в очередь.
4. Нажмите `Отправить сейчас`.
5. Проверьте, что адрес ушел из очереди в архив, а в журнале появилась запись об отправке.

## Настройка пароля администратора

Есть два варианта:

1. Через `config.json`:
   Используйте `auth.password_hash` и вставьте туда `bcrypt`-хэш.
2. Через веб-панель:
После входа откройте блок `Смена пароля`, введите текущий пароль и новый пароль. Сервис сам пересчитает `bcrypt`-хэш и сохранит его в [config.json](./config.json).

Рекомендуется:

- не использовать `auth.password` для постоянной эксплуатации
- хранить пароль администратора как `bcrypt`-хэш
- сразу заменить стандартный пароль

## Полезные поля в конфиге

- `server.address` — адрес и порт веб-сервиса
- `server.allow_remote` — если `false`, доступ разрешен только локально
- `auth.password_hash` — рекомендуемый способ хранения пароля администратора
- `auth.password_hash_env` — если хотите хранить хэш в переменной окружения
- `auth.session_secret` — секрет для подписи cookie-сессий
- `scheduler.interval_minutes` — интервал фоновой отправки
- `scheduler.run_immediately` — отправлять ли сразу после старта сервиса
- `mail.subject` — тема письма
- `mail.attachment_dir` — папка с вложениями
- `mail.attachment_paths` — явный список файлов вместо целой папки

## Где лежат данные

- [data/pending_emails.json](./data/pending_emails.json) — очередь на отправку
- [data/processed_emails.json](./data/processed_emails.json) — архив обработанных адресов
- [data/message_templates.json](./data/message_templates.json) — банк текстов
- [data/send_log.json](./data/send_log.json) — журнал событий
